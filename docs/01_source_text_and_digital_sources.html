3 Source Text and Digital Sources

3.1 The Source Text

The computational corpus described in this report is derived from Kanz al-Ummāl fī Sunan al-Aqwāl wa al-Afʿāl.

Bibliographic information

  • Title: كنز العمال في سنن الأقوال والأفعال
  • Author: علاء الدين علي المتقي بن حسام الدين الهندي (d. 975 AH)
  • Edited by: الشيخ بكري حياني
  • Indexes and verification: الشيخ صفوة السقا
  • Publisher: مؤسسة الرسالة
  • Edition: Fifth edition (1405 AH / 1985 CE)
  • Volumes: 18 (16 text volumes plus indexes)

The edition contains an author’s introduction followed by the hadith corpus. The first hadith report begins on printed page 21 and the final report appears on printed page 12,325. Hadith reports are numbered sequentially from 1 through 46,624, although some numbers are intentionally absent from the printed edition and are discussed later in this report.

Conceptually, each report consists of three principal components:

  1. Hadith number.
  2. Matn (the text of the report).
  3. Source citations identifying the original compilations from which the report was drawn.

Editorial section titles and subsection headings are interspersed throughout the printed work and are not themselves hadith reports.

3.2 Digital Source Data

Two complementary digital representations of Kanz al-Ummāl were used during development.

3.2.1 HTML/Text Archive

The first source consisted of a locally archived copy of the online Shamela edition (https://shamela.ws/book/2677). A custom Python scraper downloaded each printed page individually and stored both the original HTML and a plain-text representation. The scraper supports checkpointing, resumable execution, randomized request delays, and incremental downloading, allowing the complete corpus to be acquired reliably over multiple sessions.

The HTML archive contains all printed pages containing hadith reports and served as the primary development and validation corpus during the early stages of parser construction.

3.2.2 Shamela JSONL Database

The second source was the Shamela4 Full Database distributed through Hugging Face:

https://huggingface.co/datasets/AuthenticIlm/Shamela4_Full_DB

For this project only the Kanz al-Ummāl portion of the database was imported:

https://huggingface.co/datasets/AuthenticIlm/Shamela4_Full_DB/tree/main/06__كتب-السنة/1654__كنز-العمال-في-سنن-الأقوال-والأفعال-ط-الرسالة

Unlike the HTML archive, the database stores the text as JSON Lines (JSONL), where each physical printed page is represented by a single JSON object. The principal file used for corpus construction is pages.jsonl, which contains one record for every printed page.

The most important fields include:

Field Description
page_id Unique page identifier
book_id Shamela book identifier
part Printed volume
page_num Printed page number
sequence_num Sequential page order
body Main page text
footnotes Editorial footnotes
hints Auxiliary metadata
services_raw Additional metadata

Inspection of the source data shows that the principal textual content is stored in the body field. Editorial headings are embedded within the page text using HTML span elements and therefore require special handling during corpus construction to prevent them from being interpreted as hadith text.

3.3 Why Two Digital Sources?

The HTML archive and the structured JSONL database represent the same printed edition but serve complementary purposes.

The HTML archive proved valuable during early parser development and manual validation because it closely reflects the appearance of the online edition. The structured JSONL database subsequently became the preferred source for corpus construction because it provides explicit page-level metadata and supports efficient import into DuckDB.

Throughout development, the HTML archive continued to serve as an independent reference when investigating ambiguous parsing cases, while the JSONL database became the canonical source for automated corpus construction.
