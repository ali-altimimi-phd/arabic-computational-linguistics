6  Source Abbreviations and Narrators

7 Script 05 — Source Abbreviations and Narrator Tally

7.1 Purpose

Script 05 extends the Version 1.0 corpus from matn analysis to source-field analysis. It identifies source abbreviations, tallies their frequency across the corpus, creates a hadith-level source-abbreviation matrix, extracts preliminary narrator names from the source field, and produces audit tables for unresolved or ambiguous source tokens.

The script is designed as a conservative analytical layer. It does not attempt full isnād reconstruction. Instead, it separates what can be recognized reliably from what requires human review. This makes the current corpus useful for preliminary source and narrator analysis while preserving unresolved cases for later scholarly curation.

7.2 Outputs

The script generates nine analytical products. Each output is exported as a CSV file and mirrored into DuckDB using the same kanz_05_* table name.

Output Description
kanz_05_source_abbreviation_lookup Lookup table of abbreviation variants, canonical abbreviations, source names, status, category, and notes.
kanz_05_source_abbreviation_counts Corpus-wide frequency counts for recognized source abbreviations.
kanz_05_source_narrator_counts Frequency counts of narrator names extracted from source-field patterns.
kanz_05_source_narrator_report_audit Hadith-level audit table showing narrator spans, stop reasons, and associated source text.
kanz_05_source_token_category_counts Counts of token categories observed while parsing the source field.
kanz_05_source_unknown_review_tokens Tokens requiring review because they were not recognized as source abbreviations or standard formulae.
kanz_05_source_abbreviation_report_audit Hadith-level audit table classifying source-field recognition status.
kanz_05_hadith_source_abbreviation_matrix Hadith-by-source matrix showing source abbreviation counts per report.
kanz_05_source_analysis_run_summary Overall processing summary for Script 05.
NoteDesign note

Script 05 combines two complementary tasks: source abbreviation recognition and preliminary narrator extraction. The abbreviation analysis is lookup-driven and relatively stable. The narrator extraction is intentionally heuristic and should be treated as a first-pass index for review rather than a completed isnād analysis.

Version 1.0 includes 46,623 hadith rows. Source text is present in 38,998 rows (83.6%), while 7,625 rows (16.4%) currently lack a source field. The lookup table contains 33 canonical source abbreviations derived from 36 lookup rows.

Recognized source abbreviations occur in 31,598 reports, representing 67.8% of all rows and 81.0% of rows with source text. Across the corpus, the script identifies 48,991 source-abbreviation instances.

Preliminary narrator extraction identifies 23,875 narrator instances, corresponding to 4,283 distinct normalized narrator names. Narrator information is extracted from 22,998 reports, or 59.0% of source-present rows.

The audit table distinguishes several levels of confidence. Reports with recognized abbreviations can be used immediately for preliminary source-frequency analysis. Reports with source text but no detected abbreviation require closer inspection, since they may contain narrator-only material, non-standard formatting, or source forms not yet captured in the lookup table.

Narrator extraction is intentionally preliminary. The current rule captures phrases following عن until the end of the source field or until a recognized source abbreviation is encountered. 7.8 Interpretation

Script 05 demonstrates that the source field is already sufficiently structured to support preliminary quantitative analysis. A large majority of source-present reports contain recognized abbreviations, and the source lookup framework makes it possible to convert compact editorial symbols into analyzable source categories.

The narrator extraction results are promising but should be interpreted cautiously. They show that narrator information can be recovered at scale from recurring source-field patterns, but the output remains dependent on heuristic name-span extraction. This is useful for exploratory analysis and prioritizing manual review, but not yet sufficient for full isnād reconstruction.

The most important contribution of this stage is not simply the frequency counts themselves, but the creation of a reviewable source-analysis layer. Recognized abbreviations, narrator spans, unknown tokens, and audit statuses are all preserved in separate tables. This makes the corpus both analytically useful and incrementally correctable.

ImportantKey point

Script 05 supports preliminary source and narrator analysis, but it also identifies where human curation would have the greatest effect: unresolved source tokens, source-present rows without detected abbreviations, and narrator-name normalization.

7.9 Building on These Results

The Script 05 outputs provide the foundation for several future lines of analysis:

  • source-frequency profiling across the corpus;
  • identification of dominant source collections and compilers;
  • source co-occurrence analysis using the hadith-by-source matrix;
  • preliminary narrator frequency analysis;
  • narrator-name normalization and authority-table construction;
  • focused manual review of unresolved source tokens;
  • comparison of source patterns across thematic or structural subsets of the corpus;
  • future transmission-network analysis after additional curation;
  • eventual isnād reconstruction once source segmentation and narrator normalization are more mature.

At the current Version 1.0 stage, source abbreviation analysis is more reliable than narrator reconstruction. The best near-term use of these results is therefore exploratory source profiling and targeted corpus curation, rather than definitive claims about transmission networks.
